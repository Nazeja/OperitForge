# 洛玑表情包渲染器

将聊天消息中的 `<meme>表情名</meme>` 或 `<sticker>表情名</sticker>` 渲染为表情包图片。

## 当前版本

`0.7.2`

## 主要功能

- 支持本地表情包目录扫描：`gif / png / jpg / jpeg / webp`。
- 支持外链表情包列表，AI 仍只需要输出表情名。
- 支持 `<meme>名字</meme>` 与 `<sticker>名字</sticker>` 两种标签。
- 支持多目录模式，多目录重名时自动使用 `1- / 2-` 等数字前缀。
- 支持本地与外链重名处理：本地优先，外链自动使用 `EL-` 前缀。
- 支持按角色保存配置：不同角色可使用不同本地目录和外链列表。
- 支持自动注入系统提示词，也可以手动生成并复制提示词。
- 支持重建索引，解决新增或修改表情后缓存未刷新的问题。

## 本地表情包配置

在管理页的“表情包路径”中填写目录路径，例如：

```text
Download/sticker
```

也可以填写多个目录，每行一个：

```text
Download/sticker
Download/cat_meme
```

路径可以使用完整 Android 路径，也可以使用 `/sdcard/` 下的相对路径。

> 注意：自动扫描路径需要 Shizuku 权限。如用户没有开启 Shizuku，请使用下方的外链列表。

## 外链表情包配置

在管理页的“外链表情包列表”中填写，每行一个：

```text
哇塞: https://example.com/sticker.gif
贴贴：https://example.com/tietie.png
```

规则：

- 仅支持 `表情名: 图片URL` 或 `表情名：图片URL`。
- 支持英文冒号、中文冒号，以及冒号两侧空格。
- 支持空行。
- 支持 `#` 开头的注释行。
- URL 仅接受 `http://` 或 `https://`。

外链表情会和本地表情一起加入 `[Valid names]`。AI 输出：

```xml
<meme>哇塞</meme>
```

插件会自动将其渲染为对应外链图片。

## 重名规则

### 多个本地目录重名

如果配置多个本地目录，会自动启用数字前缀：

```text
1-贴贴
2-贴贴
```

AI 应按提示词中的有效名称输出：

```xml
<meme>1-贴贴</meme>
```

### 本地与外链重名

本地优先。若本地和外链都有 `贴贴`，则：

```text
贴贴     -> 本地表情
EL-贴贴  -> 外链表情
```

对应输出：

```xml
<meme>贴贴</meme>
<meme>EL-贴贴</meme>
```

## 角色配置

管理页顶部可以选择：

- 全局默认
- 指定角色卡

每个配置可分别保存：

- 本地表情包路径
- 外链表情包列表

按钮说明：

- `保存当前配置`：保存当前选中的全局或角色配置。
- `清除当前配置`：只清除当前选中的配置。
- `清空所有配置`：清除全部全局与角色配置。

## 提示词注入

开启自动注入后，插件会把可用表情名追加到 system prompt 中，格式类似：

```text
[Valid names]
贴贴 | 哇塞 | EL-贴贴
```

AI 只需要严格输出：

```xml
<meme>表情名</meme>
```

不要把标签包进 Markdown 或代码块。
