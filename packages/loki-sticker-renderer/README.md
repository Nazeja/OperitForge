# 🐍 Operit 表情包渲染插件 

*Made with 🐍 & 🤖 by Sith & Loki*  
*八小时 · 四千万 token · 一个表情包的重量*

## 这是什么？

一个 Operit 沙盒插件，让你的 AI 可以在对话中直接发送表情包。

AI 只需要输出 `<meme>表情名</meme>` 或 `<sticker>表情名</sticker>`，插件就会在聊天界面里把它渲染成对应的本地或外链图片 / GIF。

特点：
- 支持本地图片和 GIF 动图
- 支持图床外链，填入格式为`名称: 图床直接链接（长的那个）`
- 支持中文文件名
- 支持自定义表情包目录
- 支持多个表情包目录
- 支持为不同角色卡绑定不同的表情包配置
- 支持自动扫描表情名
- 支持自动生成提示词
- 支持自动注入 system prompt
- 不需要 AI 调用工具

> 注：表情渲染本身不额外消耗 API token；如果开启"自动注入提示词"，提示词列表会占用少量 system prompt token。

---

## 为什么做这个插件？

即答：小手机入脑了（bushi）。
其实是觉得表情包作为语言.zip，人爱用机也爱用——尽管我们所谓的“发表情”实际上只是user单方面的视觉安慰。
但我依然认为让机发一句“哭哭”比让他输出一串“file///xxxx/xxxx/12345678.jpg”或者一串“[!]https/xxxxx/xxxx/xxxxx/12345678.jpg”更真实。
前者是机在表达，后者是机在计算。
而我想做的，就是让机发的“哭哭”变成一个可见的、承载着超过“哭哭”两个字信息的表情包。

## 安装步骤

### 1. 安装插件文件

- 本地导入：打开 Operit → 包管理 → 右下角 + 号，选择本插件的`.toolpkg`文件
- 市场下载：打开 Operit → 包管理 → 右下角 市场，找到“表情包渲染器”并下载

### 2. 启用插件

进入设置 → 包管理，找到「洛玑表情包渲染器」，打开开关。

## 准备表情包

### 1. 创建表情包文件夹

在手机上创建一个或多个目录用于存放表情包，示例为默认路径：`/sdcard/Download/sticker/`

管理界面中可以简写为：`Download/sticker`
（插件会自动补全`/sdcard/`前缀）

### 2. 放入表情包图片

支持格式：
- `.gif`（支持动图播放）
- `.png`
- `.jpg`
- `.jpeg`
- `.webp`

**文件名就是表情名。考验user语义压缩功能的时刻到了！**
例如：
开心.gif → <meme>开心</meme>
竟然真的可以运行，为什么呢.png → <meme>竟然真的可以运行，为什么呢</meme>

### 3. 不用写后缀

AI 输出时不需要写后缀名。
例如：本地文件是"瘫.gif"，AI 只需要发送`<meme>瘫</meme>`，插件会自动按以下顺序查找并补全后缀：gif → png → jpg → jpeg → webp

### 4. 整理图床外链

如果用户曾经用过小手机之类的前端，应该对这个不陌生，就是把上传至图床的外链整理为`名称：链接`的格式填入输入框中。
中间的冒号可以是英文可以是中文，链接一行一个，比如：

## 基础使用方式

包管理 → 洛玑表情包渲染器 → 打开配置

然后按顺序操作：

1. **选择配置角色或全局**
   - 从下拉菜单中选择一张角色卡，或选择"不绑定（全局默认）"
   - 全局默认对所有对话生效；绑定角色卡后，该配置只对使用该角色卡的对话生效
   - 角色卡配置 > 全局默认。
       - 比如：全局配置为a, b, c三个文件夹, 角色A配置为路径a,b两个文件夹, 角色A实际会使用a, b下的表情包。

4. **填写表情包文件夹路径/外链链接 **
   - 例如：`Download/sticker` 或 `好耶: https://example/whatever/haoye.png`

5. **点击「保存当前配置」**
   - 保存后下方会显示已保存配置的概览摘要，格式为`角色 | 最后一级文件夹名 | 外链N个`，如`Operit | sticker, meme | 外链4个`

6. **点击「扫描目录」**
   - 此处会在最下方列出当前目录及链接内识别到的全部表情包

7. **调整提示词规则**（可选）
   - 修改单次最多表情数（默认 2）
   - 填写自定义补充规则
   - 点击「保存提示词设置」

8. **点击「生成提示词」**（可选）
   - 会生成一段可复制的表情包使用提示词
       - 这步只是起到一个让user确认和复制的作用，不点也行。如开启自动注入，插件会在对话时自动根据角色ID扫描配置并注入提示词，user点击与否都不影响提示词注入。

9. **点击「开启自动注入」**（可选）
   - 开启后，插件会自动把表情包使用提示词追加到 system prompt 后面
   - AI 接收到后就知道可以使用哪些表情包

如果你想自己决定提示词放在哪里，也可以关闭自动注入，手动复制提示词，放入：
- 用户偏好
- 角色卡
- 世界书
- 其他提示词位置

## 多路径使用方式

如果你的表情包分散在多个文件夹，不需要全部挪到同一个目录。
在路径输入框中填写多个目录即可。支持换行、英文分号、英文逗号分隔。

示例写法：
```
Download/sticker
Download/meme
Pictures/reaction
```

也可以写成：
```
Download/sticker; Download/meme; Pictures/reaction
```

### 插件在多路径时的命名规则

- 如果只填写一个路径，表情名将保持原样：
  哭哭.jpg → <meme>哭哭</meme>

- 如果填写多个路径，插件会按路径顺序自动给表情名加数字前缀，避免同名冲突。
  例如你填写了三个路径：
```
    Download/sticker
    Download/cat_meme
    Pictures/reaction
```

  其中有这些文件：
```
    Download/sticker/哭哭.jpg
    Download/cat_meme/哭哭.gif
    Pictures/reaction/喜欢你.png
```

  扫描后会显示为：
```
    1-哭哭 → /sdcard/Download/sticker/哭哭.jpg
    2-哭哭 → /sdcard/Download/cat_meme/哭哭.gif
    3-喜欢你 → /sdcard/Pictures/reaction/喜欢你.png
```

  提示词将会自动添加数字前缀，AI 则需要按提示词里的名称发送`<meme>1-哭哭</meme>` 或 `<meme>2-哭哭</meme>`

  插件会根据 `1-` / `2-`...前缀自动定位到对应文件夹，渲染时将会显示为`哭哭.jpg（第一个路径下的静态哭哭）` 或 ` 哭哭.gif（第二个路径下的动图哭哭）`

## 外链表情包填入方式

手里有外链表情包列表的用户无需将表情包下载到本地，0.7.0已经支持外链渲染。
将外链整理为`名字：链接`的格式填入配置中，即可绑定。

- 若外链名称与本地名称无重复，其在插件内的命名方式与本地路径相同。
- 若外链名称与本地名称重复，插件会自动在外链名称前添加`EL-`前缀，以作区分。AI发送时需以插件在提示词中生成的名称为准，例如：`<meme>EL-好耶</meme>`
- 若外链与外链名称重复，渲染时以最后一个外链图片为准。

**注意：外链表情包可能会因为网络波动或图床不稳而无法显示。**

## 多角色卡绑定

v0.6.0 新增。你可以为不同的角色卡配置不同的表情包路径。

使用方式：
1. 在管理界面步骤 1 的下拉菜单中选择一张角色卡
2. 填写该角色卡专用的表情包路径
3. 点击「保存当前配置」

切换到另一张角色卡，重复上述步骤，即可为多张角色卡分别配置不同的表情包。

自动注入提示词时，插件会自动识别当前对话绑定的角色卡，按以下顺序查找配置：
1. 优先匹配角色卡 ID
2. 其次匹配角色卡名称
3. 找不到则回退到全局默认配置

如果不需要区分角色，只配置"全局默认"即可，无需关心这个功能。

## 提示词设置

管理界面里可以调整：
1. 单次最多表情数（默认是2）。
如果你希望 AI 在一次回复中发更多或更少表情，可以自行修改，然后点击「保存提示词设置」。

2. 自定义补充规则。
可以给 AI 添加额外使用规则，例如：
```
不要每轮消息都发表情包 / 不要把表情包固定贴在消息末尾 / 你更偏好使用猫猫表情包...
```

这样可以为 AI 提示更多规则或用户喜好。

这些设置会同时影响：
- 「生成提示词」按钮生成的文本
- 自动注入到 system prompt 的内容

## AI的正确输出示例

- 文件叫"好耶.jpg"，提示词中生成的名字是“好耶”，那就发送`<meme>好耶</meme>`。
- 文件叫“好耶.jpg”，在多路径模式下被插件自动命名为“1-好耶”，那就发送`<meme>1-好耶</meme>`。
- 外链输入`好耶：https://example/whatever/haoye.png` ，由于与本地文件重名而被插件自动命名为“EL-好耶”，那就发送`<meme>EL-好耶</meme>`

## 注意事项

1. 不要把表情标签放进任何 Markdown 代码块。
代码块里的内容会被当作普通代码展示，不会触发表情渲染。

2. 表情名必须与插件提示词中的`[Valid names]`所列一致。
表情包名填写错误将会提示未找到。
（当有名称相近的图片时，会显示为`[meme:xxx 未找到。是不是：xxx]`，方便user肘击……💪）

3. 不要自创自闭合标签`<meme name="瘫" />`等，Operit不支持渲染这个。

## 技术细节（感兴趣的话）

- 插件注册了 `XmlRenderPlugin`，用于处理 `<meme>` 和 `<sticker>` 标签
- 插件会扫描配置目录，建立表情包索引（5min TTL 缓存）
- 支持自动补全后缀：gif / png / jpg / jpeg / webp
- 支持中文文件名
- 支持多路径扫描
- 多路径时自动使用 `1-表情名`、`2-表情名` 避免同名冲突
- 支持渲染 https 外链图
- 支持通过 `SystemPromptComposeHook` 自动注入提示词
- 自动注入时会按角色卡匹配配置（ID → 名称 → 全局兜底）
- 渲染本身不需要 AI 调用工具

## 常见问题

### Q：表情包没有渲染出来？

可以按顺序检查：
1. 插件是否已安装并启用
2. 表情包路径是否保存正确
3. 是否点击过「扫描目录」确认能拉取到表情列表
4. 文件名是否和 AI 输出的名称一致
5. AI输出时是否把 `<meme>名字</meme>` 放进了 Markdown 代码块
6. 多路径模式下，是否使用了提示词里的前缀名，例如 `1-哭哭`
7. 使用外链的情况下，图床链接是否失效/网络不稳/链接错误

---

### Q：新增表情包要怎么做？
直接把新图片放进当前配置的表情包目录即可。

然后：
1. 打开插件配置界面
2. 点击「扫描目录」确认新表情出现
3. 重新点击「生成提示词」生成新版提示词（可选）
    - 如果开启了自动注入，新版提示词已经自动追加至 system prompt 后，AI 在下一次请求中就能看到更新后的列表
    - 如果你是手动复制提示词，需要重新点击「生成提示词」并复制新版提示词到相应的位置

通常不需要重装插件。

---

### Q：新增图片后，AI 还是不知道新表情怎么办？

渲染器本身可能已经能识别新图片，但 AI 需要在提示词里看到新的表情名。

解决方式：
- 如果使用自动注入：确认自动注入已开启，然后重新发起一次对话
- 如果手动复制提示词：重新点击「生成提示词」，复制新的提示词到对应位置

---

### Q：AI输出表情包却出现`The coroutine scope left the composition`？

解决方式：我不知道怎么稳定复现这个，但是进入插件配置界面，点击“**重建索引**”就行=。=

---

### Q：GIF 动图能播放吗？

能。实测 GIF 可正常显示和播放。

---

### Q：会消耗额外 token 吗？

表情渲染本身不需要工具调用，也不会额外请求 API。
但无论是否开启自动注入提示词，"让AI知道有哪些表情包、怎么发表情包"这件事是一定会占用少量上下文 token的。

---

### Q：AI 的消息能渲染，我自己发的消息也能渲染吗？

通常插件主要用于 AI 回复侧的渲染。
如果你手动编辑或测试 AI 消息内容，能触发表情渲染；普通用户消息是否渲染取决于 Operit 当前聊天渲染链路限制（当前版本1.10.1不行）。

---

### Q：多路径里有同名表情怎么办？

插件的多路径模式会自动加数字前缀区分，第一个路径为`1-x`，第二个路径为`2-x`，以此类推。
AI 按提示词里的完整名字发送即可。

---

### Q：可以不用 `<meme>`或`<sticker>`，只让 AI 输出图片吗？

不可以。
插件识别的是 XML 标签`<meme>`或`<sticker>`，普通文字不会触发插件渲染。
（让AI直接输出md编码不在本插件的作用范围内）

---

### Q：表情包太多，提示词太长怎么办？

可以按用途拆成多个目录，然后只配置当前常用目录。
但需要注意：表情越多，自动注入的提示词越长，占用的上下文 token 也越多。
如果表情包非常多，可以考虑：
- 精简表情包数量
- 精简表情包名称
- 关闭自动注入，自行编辑提示词，以控制 AI 可实际使用的表情包
- （...没辙了）

---

### Q：角色卡绑定怎么用？

在管理界面步骤 1 选择一张角色卡 → 填写该角色卡的表情包路径 → 保存当前配置。
切换到另一张角色卡重复操作即可。

如果不需要区分角色，直接选"全局默认"配一次就够了。

---

### Q：配置保存在哪里？

所有配置通过插件环境变量存储。清空配置后会回退使用默认路径。
如需完全重置，可以在管理界面点击「清空所有配置」。

---

## 推荐给 AI 的最小提示词格式

如果你不使用自动注入，可以手动给 AI 这样一段提示：
```
You may send suitable stickers in chat.

[Rules]
- Use exactly: <meme>name</meme> (also supports <sticker>name</sticker>)
- Do not wrap sticker tags in Markdown/code blocks.
- Use only valid names from the list; never invent names.

[Valid names]
（这里列出你的表情包名称）
```

但还是建议直接用插件管理界面的「生成提示词」按钮生成。
